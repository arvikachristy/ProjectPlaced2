In this section, we are going to introduce all of our platform features as well as introducing the system manual and user manual. We will also mention all of the technologies we used to build a particular sections and platforms.

System Manual

Compatible Devices
Most of recent Operating System is compatible to run the server side of our platform. This includes:

  • Windows

    Windows 8 (Desktop), Windows 7, Windows Vista SP2, Windows Server 2008, Windows Server 2012 (64-bit)

  • MAC OS X

    Mac OS X 10.7.3 or later

  • Linux

    Oracle Linux 5.5+, Oracle Linux 6.x (32-bit), 6.x (64-bit), Oracle Linux 7.x (64-bit), Red Hat Enterprise Linux 5.5+, 6.x (32-bit), 6.x (64-bit), Ubuntu Linux 10.04 or later

Connectivity
For the server to run properly, make sure the device have a good connection to the internet in order to run the server properly.

Browser Support
Most browsers are supported to run this platform. However, our frontend is built from AngularJS and Ionic Framework hence the most updated version of the web browser is highly recommended. Here is the list of our recommended web browsers:
  • Safari
  • Google Chrome
  • Mozilla Firefox
  • Opera

User Manual

Run the system configurations
In order to view set up the Project Placed Team Management tool, there are two main parts of the system; the client and the server. Make sure you are connected to the internet and through the system terminal, we will start by connecting to the server
To set up the server:
  • go to the file directory where we have placed the manage.py file.
  • Type python manage.py runserver in order to run the main server of the project


Run the platform
To run the platform itself, we are going to set up the platform using Google Chrome with the localhost 8000 as the example. To do this, just write on your browser localhost:8000/trackprogress and the platform will show up.

Platform Features



The Project Banner
On the project banner, you can see the name of the company and project, as well as a small description about what the project is about. There is also an image in the middle, which can be the logo of the company/project.
This could be useful if someone is checking the page without checking previous pages related to the project which explain what the project is about. E.g. the supervisor may have a lot of projects to manage, so it could be useful to have some brief background about the project to remind them what project the page is refering to.

User Visibility + Activities
There are icons which display the users who are part of the project. Underneath that, there is the button which leads to the Gantt chart page, and the button which opens the comment box modal.
The project placed frontend already includes the user visibility icons, so we can include it in our page as well. We are using project placed CSS, so our page can be consistent with the already existing designs.



Status Section
There is a radio button with status options. The students can choose the status option they feel best represents the current status of their project. This choice will remain on their project page until the students choose a different choice. Each choice is a different colour - either green, amber or red. There is also some text, describing what the feature is for.

When interviewing a teaching assistant and the supervisor of the projects, we heard from them that they want to help assist the students with their projects as much as possible. However, it was very difficult for them to identify when projects were having difficulties. We also used the red, amber, green philosophy we had previously looked into. We also believe it is important for students to know exactly what this feature is for (which is why we have some desscription letting them now). We don't want them to feel discouraged about putting their project with major issues because that will relect poorly on them, but encouraged to select that project so they can recieve help with it.

We used Angular Material - a UI Component framework. We used it to give the radio buttons an appealing design. It worked well with our project as we are using Angular for the front end, and was easy to implement.



Gantt Chart
From the main page, we can access the Gantt Chart section. This is the page with the gantt chart on it. There are functions to search and filter the tasks as well as the ability to specify the date range you would like to view.

It makes sure the development of the project is on time and can be viewed based on timeline, milestones or sprint. You can customized the tasks to be done by dragging it around (to improve user experience) and place it on the desired date.

Dependencies between tasks are also available to make sure the user does not have to hassle in re-organizing the deadlines.

We wanted a method of tracking the tasks/ plans of the project over a period of time. We've temporarily implemented a gantt chart because in our own projects, we have used the gantt charts many times to display the development plan, and it was good to include a system that students are used to using. However, in the future, the gantt chart can be replaced with any other methods of tracking progress.

This is using Angular Gantt (which is a gantt chart component for AngularJS). We've stripped and modified some features from the original Angular Gantt - keeping the functionality but making sure users aren't confused by the large selection of unnecessary options.



Comment Box
Another feature is the comment box. When you click on the comment box button, a modal opens. In this modal, the student can write their name, as well as their message, and post it for others to see.

This wasn't included in our initial requirements, but was a bonus feature that we decided to add because. We believe from experience working with teams in university, communication is a very important key in a team project. It is possible that after viewing the page, the TA or supervisor would like to leave a comment for the students about how their project is going, so it would be useful to have such a feature.

Request Resources
Under the status radio buttons, there is a button which opens a modal. In this modal, the students can send a request to their supervisor/ teaching assistant requesting any resources that they require. They can select (from a selection of options) exactly what resource they require and how urgently they need it, as well as leave a comment with more details.

From our interview with the supervisor/ TA we heard that their experience with projects is that many times students don't ask for resources from them, even though they really require it. By including a request resources option, we want to encourage students to be more vocal if they need anything, and we want to make it easier for the supervisor to be aware of exactly what resources each project needs.

We also used Angular material for the radio buttons, since we had already implemented it in the status section. Everything is contained within an ionic modal (which is opened by an appropriate controller).



Task Table
The students can add, edit, and delete the tasks they are working on to a task table on the page. They can also search through the task list with keywords.

This is to help students organise their work (what is the due date on this task, who is going to complete this task etc), as well as letting supervisor/TA/client have an overview of all the tasks being worked on, so they can see if the team is on track.



Bi-Weekly Index
On the index page, the biweekly report section is split into two parts. On the left pane, you can see the current biweekly report being worked on. You can view and edit the report or submit it. On the right pane, you can see a list of the previous reports that have already been submitted. Clicking on one of the previous reports will take you to a page where you can view that report (but not make any changes since the page has already been submitted).

We had discussions about what would be the best way to display the information (of what the current report is and how we would display the previous reports). We started with the idea of displaying a carousel style report but came up with the conclusion that it will not be a good choice to just display a text. So we replace it with a list of reports with a calendar next to it as a sign to display the current report that the students are working in.
